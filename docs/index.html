zed-acp-ollama

Run local Ollama models in Zed's Agent panel — with native thinking blocks, model picker, and automatic project context.

curl -fsSL https://raw.githubusercontent.com/michaelknyazev/zed-acp-ollama/main/install.sh | bash

The problem

Zed's built-in Ollama integration uses HTTP streaming. Models like Qwen3 think silently for up to 400 seconds before the first content token arrives. Zed's HTTP client sees no data and kills the connection. zed-acp-ollama uses stdio JSON-RPC — no HTTP timeout can fire on a pipe. 

Zed  ──stdio──▶  zed-acp-ollama  ──HTTP──▶  Ollama
     JSON-RPC                 /api/chat
                              /api/tags

Features

🧠
Native thinking blocks

Thinking tokens stream as agent_thought_chunk — Zed renders them as collapsible blocks.

⚙️
Model picker

All models from /api/tags appear in a Zed dropdown. Switch without editing config.

📁
Project context

Reads AGENTS.md, CLAUDE.md, .cursorrules, README.md + file tree on first turn.

💬
Conversation history

Full multi-turn history per session. Context files injected once as a system message.

🔀
Per-session model

Switch models in the middle of a project without losing history.

🔭
Thinking toggle

Enable or disable streaming of thinking blocks via the thought_level dropdown.
